Anpassung von Fehlermeldungen in JSF

Definition von Fehlermeldungen

In JSF werden Fehlermeldungen automatisch generiert, wenn Validierungs- oder Konvertierungsfehler auftreten. Standardmäßig legt JSF sowohl den Inhalt als auch die Position dieser Meldungen fest. Benutzerdefinierte Validatoren und Konverter ermöglichen jedoch das Anpassen der angezeigten Nachrichten.

Häufig besteht der Wunsch, Fehlermeldungen direkt neben den betroffenen Formularfeldern anzuzeigen, anstatt sie am Anfang oder Ende des Formulars zu platzieren. Dies sorgt für eine bessere Benutzerfreundlichkeit und Übersichtlichkeit.

Anpassung der Fehlermeldungsposition

In der Java-EE-Dokumentation werden die JSF-Komponenten h:message und h:messages vorgestellt. Die Komponente h:message zeigt eine Fehlermeldung für eine spezifische JSF-Komponente an, während h:messages alle Fehlermeldungen auf einer Seite darstellt.

Um eine h:message-Komponente mit einer Eingabekomponente zu verknüpfen, wird das Attribut for verwendet. Die Platzierung der Fehlermeldung erfolgt idealerweise direkt neben oder unter der Eingabekomponente.

Stil und Darstellung von Fehlermeldungen

Nach der Positionierung der Fehlermeldungen kann deren Darstellung angepasst werden. Beispielsweise können Fehlermeldungen durch das Hinzufügen einer CSS-Klasse in einer auffälligen Farbe, wie Rot, angezeigt werden. Dies kann über das Attribut styleClass erfolgen, welches eine definierte CSS-Klasse referenziert.

Diese Methode ermöglicht eine klare Trennung von Inhalt und Darstellung, was die Wartbarkeit des Codes verbessert. Eine gut gestaltete Fehlermeldung kann nicht nur den Fehler anzeigen, sondern auch Lösungsvorschläge enthalten.

Anpassung der Fehlermeldungstexte

In vielen Fällen ist es wünschenswert, die automatisch generierten Fehlermeldungstexte anzupassen. Zum Beispiel kann es sinnvoll sein, die Feld-ID aus den Meldungen zu entfernen oder durch eine benutzerfreundlichere Beschreibung zu ersetzen. Dies kann durch das Attribut label in der entsprechenden Eingabekomponente erreicht werden.

Eigene Fehlermeldungen definieren

Um noch mehr Kontrolle über die Fehlermeldungen zu erhalten, können benutzerdefinierte Nachrichtentexte erstellt werden. Dies geschieht über Properties-Dateien, die einfache Textdateien sind und im Projekt hinterlegt werden. Diese Dateien können JSF-Systemparameter überschreiben und benutzerdefinierte Nachrichten bereitstellen.

Damit JSF diese Dateien erkennt, müssen sie in der JSF-Konfigurationsdatei faces-config.xml registriert werden. Bei modernen Projekten, die auf JSF 2.0 basieren, ist diese Datei oft nicht mehr standardmäßig vorhanden und muss gegebenenfalls manuell angelegt werden.
