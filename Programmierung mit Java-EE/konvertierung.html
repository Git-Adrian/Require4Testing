Konvertierung von Zahlen, Währungen und Datumsfeldern

Einführung in die Konvertierung

Bei der Eingabe von Daten in Onlineformularen werden die Informationen immer als Zeichenketten übertragen, unabhängig davon, ob es sich um Zahlen oder Daten handelt. Dies liegt an den zugrundeliegenden Technologien wie HTML und HTTP, die nur Zeichenkettenformate kennen. Früher musste diese Umwandlung von Hand erfolgen, was viel Aufwand bedeutete. Doch mit der Einführung von Frameworks wie JSF werden diese Schritte weitgehend automatisiert.

Standardkonvertierer in JSF

JSF bietet eingebaute Standardkonvertierer, die automatisch erkennen können, in welchen Typ eine Zeichenkette umgewandelt werden soll. Beispielsweise können Ganzzahlen und Fließkommazahlen problemlos konvertiert werden. Wenn höhere Anforderungen an die Formatierung gestellt werden, kann man die Konvertierer explizit konfigurieren, um etwa Datumsformate oder Währungsdarstellungen zu steuern.

Konvertierung von Datumsfeldern

Die Konvertierung von Datumsfeldern erfolgt in JSF durch die Komponente f:convertDateTime. Damit kann das Datum nicht nur korrekt gespeichert, sondern auch in verschiedenen Formaten angezeigt werden. Die Wahl des Datumsstils (z. B. „short“ oder „full“) ermöglicht es, das Datum je nach Bedarf unterschiedlich darzustellen.

Stil Beispiel
short 10.01.14
medium 10.01.2014
long 10. Januar 2014
full Freitag, 10. Januar 2014

Hinweis

Die Formatierung eines Datums kann auch anhand eines Musters definiert werden, um spezifische Anforderungen zu erfüllen. Ein Beispiel wäre das Muster „dd.MM.yyyy hh:mm:ss:SSS“, das ein Datum nach dem Schema „10.01.2014 15:08:14:863“ formatiert.

Konvertierung von Währungsfeldern

Für die Konvertierung von Währungswerten in JSF wird die Komponente f:convertNumber verwendet. Diese Komponente ermöglicht es, Zahlen in Währungsformaten darzustellen, wobei auch Währungssymbole und Dezimalstellen angepasst werden können. Zum Beispiel kann das Euro-Symbol durch „EUR“ ersetzt werden, wenn dies gewünscht ist.

Eigene Konvertierer erstellen

Wenn die Standardkonvertierer nicht ausreichen, können eigene Konvertierer erstellt werden. Diese benutzerdefinierten Klassen müssen das Interface javax.faces.convert.Converter implementieren und die Methoden getAsObject() und getAsString() bereitstellen. Fehlerhafte Eingaben führen zu einer ConverterException, die dem Benutzer als Fehlermeldung angezeigt wird.

Hinweis

Standardkonvertierer können durch die allgemeine JSF-Komponente f:converter in Elemente eingebunden werden. Eine Übersicht aller Konvertierer bietet die Java-EE-Dokumentation.
