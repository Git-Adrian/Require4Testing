Unified Expression Language (UEL) in JSF

Einführung in Unified Expression Language (UEL)

Nachdem Herr Koch die Grundlagen zur Umwandlung von Geschäftsobjekten in Beans und die Verwendung von Scope-Annotationen verstanden hat, widmet er sich einem weiteren zentralen Konzept von JavaServer Faces (JSF): der Unified Expression Language (UEL). Mit UEL-Ausdrücken kann Herr Koch die Verbindung zwischen JSF-Komponenten und Beans herstellen, was eine effiziente Verknüpfung von Model- und View-Komponenten ermöglicht.

Value-Expressions

Value-Expressions verknüpfen JSF-Komponenten zur Ein- und Ausgabe von Daten mit Attributen einer Bean. Das Attribut value in JSF-Komponenten wie h:outputText wird verwendet, um Werte aus einer Bean anzuzeigen oder Eingaben von Benutzern zu empfangen. Ein Beispiel zeigt, wie Attribute einer Bean über Value-Expressions in die Benutzeroberfläche integriert werden.

Method-Expressions

Method-Expressions ermöglichen die Verknüpfung von JSF-Komponenten, die Aktionen auslösen (wie h:commandLink), mit Methoden in einer Bean. Diese Ausdrücke stehen im Attribut action und ermöglichen es, Geschäftslogik auszulösen, wenn Benutzerinteraktionen auftreten. Ein Beispiel zeigt, wie Methoden durch Method-Expressions aufgerufen werden.

Verwendung von UEL für Datenmanipulation

Herr Koch plant, eine neue Seite zur Pflege von Artikeldaten zu erstellen. Dabei nutzt er JSF-Komponenten, die durch Value- und Method-Expressions mit den Attributen und Methoden der Artikel-Objekte verknüpft werden. Die Darstellung und Manipulation von Artikeldaten erfolgt über ein Formular, das JSF-Komponenten wie h:inputText und h:outputLabel verwendet.

Controller-Bean und JSF-Komponenten

Zur Verarbeitung von Benutzereingaben und Navigation benötigt Herr Koch eine Controller-Bean. Diese Bean übernimmt die Aufgabe, Eingaben zu verarbeiten und zwischen den JSF-Komponenten und der Bean zu vermitteln. Eine separate Controller-Bean für jede HTML-Seite ist eine bewährte Methode. Ein Beispiel zeigt den Controller ArtikelController, der die Artikelpflege ermöglicht.

Verwendung des Singleton-Musters

Um auf das gleiche Sortiment über verschiedene Controller hinweg zuzugreifen, verwendet Herr Koch das Singleton-Muster. Die Bean Shop wird mit @ApplicationScoped annotiert, sodass alle Controller die gleiche Instanz nutzen können. Dies ermöglicht eine konsistente Verwaltung des Sortiments über die gesamte Anwendung hinweg.

JSF-Komponenten für Formulare

Für die Eingabe und Bearbeitung von Daten verwendet Herr Koch JSF-Komponenten wie h:panelGrid für die tabellarische Anordnung und h:inputText für die Eingabe von Werten. Die Verknüpfung von JSF-Komponenten mit Beans erfolgt über Value-Expressions, und die Beschriftungen werden mit h:outputLabel realisiert. Ein Beispiel zeigt, wie diese Komponenten im Formular zur Artikeldatenpflege verwendet werden.

Weitere Möglichkeiten der UEL

UEL bietet zusätzliche Funktionen, wie die Auswertung von booleschen Ausdrücken und die Verwendung von Operatoren. Zum Beispiel kann die Sichtbarkeit von Komponenten mit dem Attribut rendered gesteuert werden. Auch komplexere Ausdrücke, wie die Formatierung von Preisen oder der Zugriff auf Array-Elemente, sind möglich. Beispiele zeigen, wie diese erweiterten Funktionen der UEL genutzt werden können.
