Seitennavigation mit JSF-Anwendungen

Einführung in die Seitennavigation

In JSF-Anwendungen reicht es oft nicht aus, nur einzelne Seiten zu gestalten. Es ist ebenso wichtig, eine nahtlose Navigation zwischen verschiedenen Seiten zu ermöglichen. Ein typisches Beispiel ist die Navigation von einer Login-Maske zu einer geschützten Seite, wie etwa einer Artikelübersicht. JSF bietet hierfür spezialisierte Techniken, um diese Anforderungen zu erfüllen.

Navigation in Webanwendungen

Während in HTML die Navigation hauptsächlich über einfache Hyperlinks erfolgt, reicht dies für komplexere Webanwendungen oft nicht aus. In JSF ist die Navigation häufig von der Anwendungslogik abhängig. Geschäftsregeln bestimmen, ob ein Benutzer zu einer bestimmten Seite navigieren darf und welche Seite ihm als nächstes angezeigt werden soll.

In der entsprechenden Phase des JSF-Lebenszyklus wird die Navigation durch Methoden gesteuert, die in der Anwendungslogik aufgerufen werden. Dies erlaubt eine flexible und kontrollierte Weiterleitung basierend auf den Eingaben und Aktionen des Benutzers.

Implementierung einer Login-Funktion

Um die Navigation in einer JSF-Anwendung zu steuern, kann eine einfache Login-Funktion implementiert werden. Dabei werden Benutzername und Passwort mit einer Liste vorgefertigter Benutzer verglichen. Ist der Vergleich erfolgreich, wird der Benutzer zur geschützten Seite weitergeleitet, andernfalls bleibt er auf der Login-Seite.

Dies wird durch eine Methode im Controller erreicht, die die Eingabedaten prüft und abhängig vom Ergebnis die entsprechende Seite zurückgibt.

Erstellung der Login-Seite

Die Login-Seite in JSF kann einfach gestaltet werden. Sie enthält eine Eingabemaske für den Benutzernamen und das Passwort sowie eine Schaltfläche, die den Anmeldevorgang auslöst. Diese Schaltfläche ist mit einer Methode im Controller verknüpft, die die eingegebenen Daten überprüft und bei Erfolg zur nächsten Seite navigiert.

Konfiguration der Startseite

Beim Erstellen eines JSF-Projekts wird oft eine Standard-Startseite eingerichtet. Diese kann jedoch angepasst werden, um beispielsweise die Login-Seite als neue Startseite festzulegen. Dazu muss die Konfiguration in der Datei web.xml geändert werden, die in den „Configuration Files“ der Entwicklungsumgebung zu finden ist.

Überprüfung der Login-Funktion

Nach der Implementierung der Login-Funktion sollte diese gründlich getestet werden. Dabei ist es wichtig, sowohl gültige als auch ungültige Kombinationen aus Benutzername und Passwort zu testen, um sicherzustellen, dass die Navigation korrekt funktioniert.

Bei ungültigen Eingaben sollte der Benutzer auf der Login-Seite bleiben und ihm eine entsprechende Fehlermeldung angezeigt werden. Diese Funktion kann im nächsten Schritt durch eine Validierung der Eingabedaten und die Ausgabe passender Fehlermeldungen ergänzt werden.
