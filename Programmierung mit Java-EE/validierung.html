Validierung von Benutzereingaben

Pflichtfeldvalidierung

Eine Pflichtfeldvalidierung stellt sicher, dass bestimmte Felder nicht leer bleiben. JSF bietet dazu das Attribut required, das auf „true“ gesetzt wird, um sicherzustellen, dass das Feld ausgefüllt werden muss. In der Artikelverwaltung wird diese Validierung zunächst für den Artikelnamen eingeführt.

Beim Testen stellt sich heraus, dass ohne Eingabe eines Namens eine automatisch generierte Fehlermeldung angezeigt wird, die den Benutzer darauf hinweist, das Feld auszufüllen, bevor er fortfahren kann.

Formatvalidierung

Die Formatvalidierung kommt zum Einsatz, wenn die Eingabe einem bestimmten Muster entsprechen muss. Ein gängiges Beispiel ist die Überprüfung des Dateinamens eines Artikelbildes, der bestimmten Regeln folgen sollte, wie etwa der Verwendung einer gültigen Dateiendung (jpeg, jpg, gif oder png). JSF bietet hier die Komponente f:validateRegex, mit der ein regulärer Ausdruck zur Validierung genutzt werden kann.

Nach Implementierung der Formatvalidierung zeigt ein fehlerhafter Dateiname eine Fehlermeldung an, die auf das nicht angepasste Regex-Muster hinweist.

Fachliche Validierung

Fachliche Validierungen überprüfen, ob Eingaben bestimmten fachlichen Anforderungen entsprechen. Zum Beispiel müssen Artikelnummern mindestens vierstellig sein. Für solche Überprüfungen bietet JSF verschiedene Komponenten wie f:validateLongRange für Ganzzahlen oder f:validateDoubleRange für Fließkommazahlen an. Diese Validierungen werden durch die Attribute minimum und maximum konfiguriert.

Die erzeugte Fehlermeldung weist den Benutzer darauf hin, dass der eingegebene Wert den zulässigen Wertebereich überschreitet oder unterschreitet.

Benutzerdefinierte Validierung

Für komplexere Validierungen, die nicht mit den Standardvalidierern abgedeckt werden können, lassen sich benutzerdefinierte Validatoren implementieren. Ein Beispiel dafür ist die Überprüfung eines global eindeutigen Barcodes (GTIN), der mithilfe einer Prüfziffer auf Richtigkeit geprüft wird. Hierfür wird eine eigene Validatormethode implementiert, die bei einem Fehler eine ValidatorException auslöst.

Durch die Implementierung einer solchen Methode ist sichergestellt, dass auch spezielle Anforderungen an die Validierung, wie die Überprüfung der GTIN-Prüfziffer, korrekt durchgeführt werden.
