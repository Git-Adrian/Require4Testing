Ausgewählte Gruppen von NoSQL-Systemen

Key-Value-Systeme

Das Prinzip der Key-Value-Systeme ist simpel: Ein Schlüssel (engl. key) wird mit einem Wert (engl. value) verknüpft, wobei der Wert selbst komplexe Datenstrukturen wie Listen oder Mengen enthalten kann. Diese Systeme ermöglichen eine schnelle und effiziente Organisation großer Datenmengen ohne schemabedingte Abhängigkeiten zwischen Datensätzen. Die Formulierung von Abfragen wird jedoch häufig durch die API des Herstellers beschränkt.

Abbildung: Datenorganisation in Key-Value-Systemen

Typische Vertreter

  • Redis
  • Chordless
  • Riak

Wide Column Stores

In Wide Column Stores werden Daten in spaltenorientierten Tabellen gespeichert. Im Gegensatz zu relationalen Datenbanken, die zeilenweise speichern, werden hier alle Daten einer Spalte direkt hintereinander abgelegt, was schnelles Lesen und Ändern ganzer Spalten ermöglicht. Diese Struktur ist ideal für analytische Anwendungen, wie sie im Data-Warehouse-Umfeld zum Einsatz kommen.

Abbildung: Datenorganisation bei Wide Column Stores

Typische Vertreter

  • HBase
  • Cassandra
  • Amazon SimpleDB

Document Stores

Document Stores speichern keine traditionellen Dokumente, sondern strukturierten Text in Formaten wie JSON oder XML. Diese flexible Struktur ermöglicht es, Daten ohne festes Schema zu speichern. Allerdings muss die Konsistenz der Daten durch die Anwendung sichergestellt werden.

Abbildung: Schema von Document Stores

Typische Vertreter

  • CouchDB
  • MongoDB

Graphdatenbanken

Graphdatenbanken verwenden Graphen als Datenstruktur zur Organisation der gespeicherten Informationen. Diese Struktur eignet sich besonders für vernetzte Informationen wie soziale Netzwerke oder Verkehrsnetze. Durch die Knoten und Kanten, die typisiert werden können, lassen sich Beziehungen zwischen Daten effizient abbilden und schnell durchqueren.

Typische Vertreter

  • AllegroGraph
  • Neo4j
