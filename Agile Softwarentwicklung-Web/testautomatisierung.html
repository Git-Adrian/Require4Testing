Testautomatisierung

Einführung

Um häufige Änderungen an der Software effizient zu testen, sind Regressionstests erforderlich. Diese Tests müssen automatisiert werden, um den Aufwand für die Testdurchführung und -dokumentation zu minimieren.

Automatisierung von Unit-Tests mit JUnit

Unit-Tests sind weit verbreitet und zielen darauf ab, alle Anweisungen, Bedingungen und Pfade in Systemkomponenten zu testen. Die Automatisierung dieser Tests bietet Vorteile wie:

  • Reproduzierbarkeit: Unit-Tests sollten reproduzierbar sein und durch festgelegte Testdaten unabhängig von anderen Tests funktionieren.
  • Unabhängigkeit: Tests sollten unabhängig sein und keine funktionalen Abhängigkeiten aufweisen. Jeder Test muss die benötigten Vorbedingungen selbst herstellen.
  • Regressionsfähigkeit: Tests müssen nach Änderungen am Code weiterhin erfolgreich sein, um die Funktionalität sicherzustellen.

JUnit

JUnit ist ein beliebtes Tool für die Automatisierung von Unit-Tests in Java. Die Tests werden in derselben Umgebung wie der Quellcode programmiert, um die Übersichtlichkeit zu wahren. Ein Testfall wird durch Annotationen wie @Test definiert, und Assertions wie assertEquals überprüfen die Ergebnisse.

@Test
public void testBerechneGesamtwert() {
    // Testdaten
    double kurs = 100.0;
    int anzahl = 10;
    
    // Methode aufrufen
    double gesamtwert = wertpapierRechner.berechneGesamtwert(kurs, anzahl);
    
    // Überprüfung
    assertEquals(1000.0, gesamtwert, 0.001);
}

Automatisierte Integrationstests mit Mockito

Integrationstests prüfen das Zusammenspiel von Komponenten. Bei fehlenden Komponenten verwenden wir Platzhalter wie Treiber und Mock-Objekte. Mockito ist ein Tool zur Erzeugung dieser Mock-Objekte und hilft bei der Simulation von Schnittstellenverhalten.

Mockito.when(mockDAO.getKurs()).thenReturn(100.0);

Automatisierung von System- und Systemintegrationstests

Diese Tests prüfen integrierte Systeme und deren Umgebung. Hier können auch Benutzerschnittstellen automatisiert werden, wobei Werkzeuge wie Selenium oder SikuliX zum Einsatz kommen.

Automatisierung fachlicher Tests

Für fachliche Tests, die Benutzerschnittstellen beinhalten, können Automatisierungswerkzeuge verwendet werden. Diese müssen jedoch sorgfältig gepflegt werden, um Stabilität bei Änderungen an der Benutzeroberfläche zu gewährleisten.

Testdatenmanagement

In größeren Projekten ist es wichtig, Testdaten getrennt von Testfällen zu verwalten. Dies erleichtert die Pflege und Anpassung der Testdaten und reduziert die Wartungskosten.
